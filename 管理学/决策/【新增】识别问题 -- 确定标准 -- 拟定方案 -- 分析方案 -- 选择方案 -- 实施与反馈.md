# 识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈

## 写在前面

　　本文讨论「管理学」下，「决策的过程」的「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」话题。
　　「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」属于「知道容易、做对很难」的典型话题。

## 正文

　　**核心观点：把 识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈 放回具体情境，才能看见它真正的解释力。**
　　在「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」这个主题里，最常见的偏差是把 目标 当成静态结论，而忽略 执行 与 反馈 的联动。
　　换句话说，识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈 不是一条孤立知识点，而是一套会随场景变化的判断规则。
　　下面按「是什么-为什么-怎么用」拆解「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」，避免只停留在口号层面。

　　如果「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」当前没有下级分支，可以用「三问法」自检：
　　1. 它在解释「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」时，解决的核心问题是什么？
　　2. 它与 协同 的上位概念相比，新增了哪些判断维度？
　　3. 在哪些条件下，「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」这套判断会失效？

> 　　某学习者在应用「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」时，只复制步骤，不验证前提。短期看似有效，长期却不断返工。

　　常见误区有两类：一是「把局部经验当成普遍规律」，二是「重技巧轻结构，导致重复犯错」。
　　<u>对「识别问题 -> 确定标准 -> 拟定方案 -> 分析方案 -> 选择方案 -> 实施与反馈」而言，判断是否真正掌握，不看能否复述定义，而看能否在新场景中给出一致、可解释、可复盘的选择。</u>
